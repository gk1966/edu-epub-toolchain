Manual · capabilities & limitations

Edu EPUB Creator — reference manual

What the authoring tool can do, how it exports, and its current limitations — stated honestly.

▶ Open the Creator

Capabilities

Editing
  • Visual editor, XHTML source view and live preview.
  • Headings, bold/italic, lists, links, text alignment and two‑column layouts.
  • Editable image alternative text for accessibility.
Reusable block library
  • About 40 ready blocks: callouts (note/tip/warning/important), definitions, key terms, worked examples, quizzes (multiple choice / true‑false / fill‑in), checklists, flashcards, tables, image and gallery blocks, buttons, call‑to‑action, steps, timeline, section layouts and more.
Import
  • Word (.docx) → chapter, keeping structure and embedding images.
  • Markdown and plain text → chapter.
  • Existing EPUB → its chapters and images, with references rewritten to assets.
  • HTML page or full HTML package (with rewritten local paths) → interactive chapter.
Media & interactivity
  • Import images, audio and video as organised assets.
  • Embed interactive HTML and 3D/WebGL content as chapters.
Standards‑based export
  • Structured EPUB 3 package (mimetype, container, package document).
  • Navigation document and EPUB 2 NCX for backward compatibility.
  • Dublin Core and conservative schema.org accessibility metadata; organised asset folders and styles. Metadata is advisory and requires review before distribution.
  • Scripted HTML spine items are linked to generated XHTML fallback summaries for reading systems that do not support their media type.
  • Optional cover image.
Workflow & interface
  • Local‑first and private: nothing is uploaded; autosave plus save/open of a project file.
  • About 50 book languages for metadata; bilingual interface (English / Greek).

Limitations

  • Import preserves semantic formatting (headings, bold/italic, lists, tables, images), not pixel‑perfect visual layout — which is the appropriate, reflowable behaviour for EPUB.
  • Markdown images that use a local file path need the image added separately as an asset; images from Word and EPUB are embedded automatically.
  • No PDF import: converting PDF to clean, reflowable EPUB is lossy, so it is intentionally omitted.
  • Interactive/scripted chapters may not run in all third‑party readers (they do run in the bundled Edu Ebook Reader).
  • For strict conformance, validate the exported file with EPUBCheck separately.
  • PWA installation and offline caching require HTTPS, or localhost during development. Opening the files through file:// or ordinary insecure remote HTTP is not equivalent to installing the app.
Why this matters: the tool is designed so that teachers who are not familiar with EPUB internals can still produce structured, interactive EPUB 3 packages with accessibility‑oriented authoring support. Formal validation and accessibility review remain separate steps.
