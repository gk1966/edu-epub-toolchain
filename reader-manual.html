Manual · capabilities & limitations

Edu Ebook Reader — reference manual

What the reader can do, how it is built, and its current limitations — stated honestly.

▶ Open the Reader

Capabilities

Formats it opens
  • EPUB and EPUB 3 (XHTML chapters, media, navigation).
  • PDF (rendered in a canvas; no PDF JavaScript executed).
  • Plain text, Markdown, HTML/XHTML.
  • DOCX and ODT (text extraction), FB2 (text), CBZ (image pages).
Interactive & 3D content
  • Runs scripted EPUB 3 content and embedded HTML.
  • Renders WebGL / 3D scenes (e.g. molecular models) with rotate, zoom and fullscreen inspection.
  • Plays embedded audio and video.
Privacy & security
  • Files are opened locally through browser APIs; the application contains no book‑upload function.
  • No application accounts or analytics are included. A hosting provider may still process ordinary connection logs under its own policy.
  • EPUB content runs in a sandbox that blocks external network access and disallows iframe, object, embed and form submission.
  • Detects DRM / encryption markers (rights.xml, encryption.xml, Readium LCP) and refuses such files; contains no decryption.
Reading experience & accessibility
  • Adjustable font, size, theme, line spacing, text width, letter spacing and reduced motion.
  • Basic read‑aloud via the browser speech API where available.
  • Remembers the last reading position per file; bilingual interface (English / Greek).
  • Installable, offline‑capable Progressive Web App.

Limitations

  • It is not designed to open DRM‑protected publications; this is an intentional scope decision, not a bug.
  • It is not a full commercial EPUB engine; some non‑EPUB formats (DOCX, ODT, FB2, Markdown) are basic display only.
  • 3D content depends on WebGL support and device performance.
  • It does not support Kindle formats (AZW/KFX).
  • Accessibility follows good practice but has not yet been independently audited against WCAG 2.2 AA.
  • Verified through a documented manual test procedure; it does not yet ship a large automated test suite.
  • PWA installation and offline caching require HTTPS, or localhost during development. Opening the files through file:// or ordinary insecure remote HTTP is not equivalent to installing the app.
Scientific/technical note: EPUB 3 is an open, web‑based publication standard (W3C). Whether interactive objects run depends on the reading system; many general‑purpose readers restrict scripting and WebGL, which is exactly the gap this reader addresses.
